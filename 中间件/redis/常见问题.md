4. 数据库缓存不一致问题
    1. 为什么发生? 更新数据时写入数据库和删除redis 不是原子的操作？
        1. 先写数据库，再删redis        如果写入数据库成功,但删除redis失败了
        2. 先删除redis,再写数据库       如果删除成功,但在写入(事务未提交)之前,另一个线程发出读请求,会将过期数据重新读入缓存
    2. 解决方案
        1. 延时双删 
            1. 先删除缓存
            2. 写入数据库
            3. 休眠xxx毫秒(保证事务提交)
            4. 再次删除缓存
        2. 异步更新(订阅binlog 更新redis,可以使用阿里的canal)
